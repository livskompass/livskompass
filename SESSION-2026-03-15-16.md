## What Was Done

### 1. Style Separation — CMS UI vs Page Content
- All brand colors converted from hardcoded hex to CSS variable-backed RGB channels
- Tailwind preset uses `rgb(var(--color) / <alpha-value>)` for opacity support
- Admin chrome switched from `stone-*` (brand) to `zinc-*` (CMS neutral)
- Editor chrome uses `--editor-*` CSS variables
- CMS overlays on public site use `blue/zinc` (not brand colors)
- Result: rebranding = change CSS vars only, CMS stays untouched

### 2. Structural Fixes
- **Sticky CMS sidebar** — stays fixed while content scrolls
- **Empty CMS content pages** — auto-converts legacy WordPress HTML to Puck blocks
- **Draft/publish separation** — autosave writes to `draft` column, publish is manual
- **Fullscreen hero preset** — 100vw × 100svh with video/image background support
- **beforeunload warning** — prevents accidental data loss on tab close

### 3. Settings & Editing UX Overhaul
- **Fixed blockType label vs key mismatch** — root cause of ALL broken settings popovers
- **SettingsPopover** — reads fields via `(puckConfig as any)` to bypass Puck Config type stripping
- **resolveFields support** — Hero shows only relevant fields per preset
- **Settings cog on every block** — popover handles empty state gracefully
- **Removed visual controls row** — all options consolidated in one settings popover (400px wide)
- **Page picker** — searchable dropdown for all link fields (replaces typing slugs)
- **Entity settings drawer** — labeled "Page settings" / "Course details" etc.
- **Course status editable** — admin can change active/full/completed/cancelled
- **Homepage toggle** — "Set as homepage" button in page settings + badge in pages list

### 4. Data-Bound Block Badges
- **EditItemBadge component** — blue "Edit page/post/course/product" badge on data-bound cards
- Applied to: PageCards, CourseList, ProductList, PostGrid
- Links to CMS editor (opens in new tab), works on both admin and public site
- Admin URL auto-detection (localhost:3001 vs production)

### 5. Array Item Controls
- **AddItemButton** — visible "+ Add item" button at bottom of all 8 array blocks
- **ArrayItemControls** — move up/down + delete per item on hover
- **ArrayDragProvider** — pointer-event drag-to-reorder with floating ghost

### 6. Block Editing Surface Classification
- `block-editing-surface.ts` — every block classified as `inline` or `linked`
- Pencil icon for inline blocks, external-link icon for data-bound blocks

## Final Fixes (2026-03-16 morning)

### Publish Fix (CRITICAL)
- **Root cause**: `content_versions` table missing from production D1 database
- `snapshotBeforePublish()` crashed on every publish attempt with "table not found"
- Fixed: created table + indexes via `wrangler d1 execute`
- Added try/catch + validation to page PUT endpoint

### Cache Fix
- Reduced API cache from `max-age=60` to `max-age=5` across all public routes
- Published changes now appear on production within ~5 seconds

### Hero Fullscreen Fix
- Changed `100svh` to `calc(100svh - 80px)` to account for nav bar height
- Hero now fits exactly below navigation without overflow

### Hero Unified Styling (Final)
- Moved shared style helpers (`isDark`, `textPrimary`, `btnPrimary`, `headingStyle`) BEFORE all preset branches
- ALL 6 presets now use identical styling variables — zero hardcoded colors
- Fullscreen, full-image, minimal, split, centered all share same typography and color logic
